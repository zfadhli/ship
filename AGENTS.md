# AGENTS.md

## Project Overview

**@zfadhli/ship** — Release automation library. Converts the classic `scripts/release.sh`
bash workflow into a programmatic, function-based TypeScript API with an optional CLI.

Converts conventional commits since the last tag into a version bump, changelog entry,
git tag, and GitHub Release — all from a single `release()` call.

## Setup Commands

```bash
bun install              # install dependencies
bun install -d <pkg>     # add dev dependency
bun install <pkg>        # add runtime dependency
bun install --frozen-lockfile  # CI install (exact versions)
```

## Development Workflow

```bash
bun run dev               # tsdown --watch (rebuild on file change)
bun run typecheck         # tsc --noEmit (type-check src/ + test/)
bun run lint              # biome check . (lint + format check)
bun run lint:fix          # biome check --write . (auto-fix)
bun run format            # biome format --write .
bun test                  # run all tests
bun test --watch          # run tests in watch mode
bun run ci                # full pipeline: typecheck → lint → test → build
```

## Testing Instructions

- Framework: `bun:test` (built into Bun, no extra config)
- Test files: `test/*.test.ts`
- Import source from `../src/core/index.ts` (Bun resolves `.ts` natively)
- Use `describe`/`it`/`expect` from `bun:test`

## Code Style Guidelines

### API Design
- **Prefer function-based over classes** — pure functions, composable utilities
- **Minimal API surface** — expose only what's needed from `src/core/index.ts`
- **Progressive complexity** — `release()` for simple cases, individual functions for power users

### Code Style
- **Clean, readable, pragmatic** — avoid cleverness
- **Strong TypeScript, strict mode**

### Structure
- `src/core/` — library source (the programmatic API)
- `src/cli/` — CLI binary (uses `@zfadhli/koko-cli`)
- `test/` — tests
- **Imports**: use `.ts` extension in import paths, `import type` for type-only imports
- **Internal modules**: `src/core/internal/` — mark with `@internal` JSDoc

## Build

```bash
bun run build    # tsdown → dist/core/index.mjs + dist/cli/index.mjs + .d.mts + sourcemaps
```

## CLI Usage

```bash
bun run src/cli/index.ts release     # run the full release flow
bun run src/cli/index.ts status      # preview next release
bun run src/cli/index.ts --help      # all commands
```

## Versioning and Release

Version bumps are derived from conventional commits since the last tag.

| Conventional commit type | Bump |
|---|---|
| `feat` + `BREAKING CHANGE` | major |
| `feat` | minor |
| `fix`, `perf`, `refactor`, `docs`, `chore`, `style`, `test` | patch |

## Git Hooks (Lefthook)

- **pre-commit**: `biome check --staged --write --unsafe` + `tsc --noEmit` (staged files only)
- **pre-push**: `bun test` + `bun run build`

## Important Notes

- **Bun is the package manager** — don't use npm/pnpm. Keep `bun.lock` in sync
- **Never commit automatically** — stage, analyze, and present the commit message, but user runs `git commit`
- **`dist/`** — generated by `bun run build`, do not modify directly
- **`package.json` `private` field** — defaults to `true`. Set `false` before publishing to npm
- **`CHANGELOG.md`** — auto-generated by `release()`, do not edit manually
- **`tmp/`** — gitignored, use for scratch files
